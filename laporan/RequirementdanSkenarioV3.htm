Project Description

Sistem Small Business Project ini merupakan sistem aplikasi yang menyediakan informasi penjualan makanan, minuman, layanan print/fotocopy berbasis web. Di sini kami menyediakan 3 fitur utama, yaitu order, delivery dan payment. Fitur order berfungsi sebagai sarana untuk user dalam memesan atau berbelanja pada web ini. Fitur delivery berfungsi sebagai jasa antar dari minimarket yang terkait. Fitur payment berfungsi sebagai sarana pembayaran bagi para pembeli/user.

 

Requirement Summary (Requirement System yang akan dibangun)
Key Requirement :

1.       Detail makanan dan minuman (termasuk dalam fungsi order)

Adanya detail makanan dan minuman ini, untuk memudahkan user mengetahui deskripsi makanan/minuman yang akan dipesan tersebut.

 

2.       Jenis Makanan (termasuk dalam fungsi order)

 

3.       Jenis Minuman (termasuk dalam fungsi order)

4.       Rating (termasuk dalam fungsi order)
Untuk memudahkan penilaian pemesan ketika sedang melakukan order, sehingga user tahu menu yang sering dipesan dan menjadi rekomendasi

5.       Warna (tampilan user interface)

Kesesuaian warna dengan topik, sangat berpengaruh atas kesan pertama user saat melihat tampilan.

 

6.       Harga (termasuk dalam fungsi Order)

Harga menjadi sorotan yang termasuk penting untuk user dalam melakukan pemesanan, Karena jika harga yang

 

7.       Searching (termasuk dalam fungsi order)

memudahkan user untuk mencari makanan ataupun minuman tanpa harus melihat keseluruhan menu makanan /minuman yang ada dengan fitur seperti search by nama makanan ataupun search by minuman, dll.

 

8.       Paket makan (termasuk dalam fungsi order)

User bisa mudah dalam memilih menu yang sudah disediakan oleh penjual, dalam bentuk paket (Makanan + Minuman) karena biasanya harga menu paket lebih murah dan lebih praktis untuk user.

 

9.       Status Ketersediaan (termasuk dalam fungsi order)

Memudahkan user untuk melihat stok ketersediaan makanan. Jadi, jika stok habis, maka user akan mengganti pesanannya.

 

10.   Contact Person (termasuk dalam fungsi delivery)

Adanya CP ini, agar memudahkan kebutuhan user dalam mengkonfirmasi pesanannya yang jika pesanan tersebut mengalami keterlambatan dalam mengantarkan pesanannya.

 

11.   Bukti pembayaran (termasuk dalam fungsi payment)

 

12.   Status pengiriman (termasuk dalam fungsi payment)

Memudahkan user untuk melihat status pengiriman makanan atau minuman yang dipesan, apakah sudah dikirim/tidak.

Design Description

Dari 4 design interface yang kami buat, kami memilih rancangan seperti pada beberapa gambar berikut:

Tampilan awal dari sistem small business project. User dapat menggunakan fitur pencarian dan memilih makanan berdasarkan kategori.

Tampilan ketika user memilih salah satu kategori dari tiga kategori yang tersedia.

Tampilan ketika user memilih detail. User dapat melihat deskripsi singkat dari produk tersebut, rating, harga, dan dapat memilih jumlah produk yang akan dipesan.

Tampilan ketika user telah menambahkan pesanan nya ke keranjang maka pada pojok kanan atas akan muncul banyaknya pesanan yang akan dipesan.

Tampilan ketika user menekan button keranjang di sudut kanan atas. User dapat menampilkan jenis produk, jumlah dan harga dari produk yang akan dipesan.

Tampilan ketika user selesai memilih pesanan yang akan dipesan. User mengisi data pengiriman berupa nama, pekerjaan, alamat pengiriman dan kontak yang dapat dihubungi.

Tampilan ketika user selesai mengisi data pengiriman dan melanjutkan pesanan ke tahapan pengiriman. Design disamping menunjukkan metode pembayaran tunai. Pembayaran dilakukan ketika barang sudah diterima.

Tampilan ketika user selesai mengisi data pengiriman dan melanjutkan pesanan ke tahapan pengiriman. Design disamping menunjukkan metode pembayaran transfer ke rekening tertentu milik small business project. Pembayaran dilakukan sebelum barang dikirim.

Tampilan ketika user sudah melengkapi proses pemesanan dan pembayaran via transfer. User harus mengunggah bukti pembayaran. Setelah bukti pembayaran diunggah maka barang yang dipesan dapat dikirim.

Tampilan ketika user telah menyelesaikan semua proses pemesanan, pembayaran dan pengiriman. User dapat kembali ke halaman utama dengan menekan button Kembali Pesan.

Tampilan untuk kurir pengantar pesanan pelanggan. Kurir dapat melihat daftar pesanan yang dipesan oleh pelanggan.

Tampilan untuk kurir pengantar pesanan pelanggan. Kurir dapat melihat detail pesanan dari pelanggan dan detail pengiriman berupa alamat pengiriman dan kontak pelanggan.

 

Terdapat 4 buah icon. Icon pertama (dari kiri) berfungsi untuk menghubungi pelanggan via telepon. Icon kedua berfungsi untuk menghubungi pelanggan via SMS. Icon ketiga sebagai informasi status pembayaran pesanan. Icon keempat sebagai informasi status pengiriman pesanan.


Racangan ini dipilih karena kejelasan yang ada pada fitur sesuai dengan tahapan yang sebenarnya. Fitur yang ada pada rancangan terpilih ini, antara lain :
- Detail deskripsi makanan dan minuman
- Metode Pembayaran : Bayar di tempat dan Transfer
- Search: User dapat mencari nama makanan / minuman
- Rating: User dapat mengetahui produk yang paling diminati oleh pembeli yang lain.

Fungsional

-          User dapat memesan produk

-          Produk dapat dikirim ke gedung kuliah pelanggan atau sekitar kampus

-          Kurir dapat mengetahui daftar pengiriman produk

-          User dapat memilih produk yang diinginkan

-          User dapat menentukan jumlah produk

-          User dapat menentukan lokasi yang dituju

-          User dapat menentukan metoda pembayaran yang diinginkan

Non-Fungsional

-          User dapat mencari produk yang diinginkan

-          Fitur Help untuk membantu pelanggan dalam menggunakan sistem

-          Fitur Kritik dan Saran untuk menampung kritik dan saran dari pelanggan baik secara sistem maupun bisnis dari small business project

-          Fitur Sorting untuk membantu pelanggan dalam memilih produk

Usability Criteria

Dari hasil wawancara kami dengan user, user lebih memilih rancangan tersebut karena beberapa hal berikut:

-          Konten dari setiap produk lebih detail

-          Sesuai dengan perilaku di dunia nyata

-          Tahapan yang dilakukan user lebih detail dari desain lainnya

 

Skenario Rancangan (Order dan Payment)

Aktor

Sistem

1.   User membuka halaman front end untuk melakukan order

 

 

2.   Menampilkan pilihan menu seperti menu makanan, minuman, jasa serta menampilkan gambar - gambar keseluruhan menu yang tersedia

3.   User memilih menu makanan / minuman yang tersedia di daftar menu

 

 

4.   Menampilkan gambar menu makanan / minuman, yang terdapat tombol Order dan Detail

5.   User memilih tombol Detail untuk mengetahui makanan / minuman tersebut

 

 

6.   Halaman detail ini, akan menampilkan deskripsi makanan ataupun minuman, status ketersediaan, jumlah order, dan icon/button keranjang.

7.   User meng-klik button dengan icon keranjang

 

 

8.   Menampilkan tabel data pesanan (Nama Produk, Qty, Harga Satuan, dan Subtotal) berserta buttonLanjutkan Pemesanan.

9.       User meng-klik buttonLanjutkan Pemesanan

 

 

10.   Menampilkan form data pengiriman yang harus diisi oleh user dan terdapat button Lanjutkan Pemesanan

11.   User melakukan isian data pengiriman

 

12.   User meng-klik button Lanjutkan Pemasanan ketika isiannya sudah diisi semua.

 

 

13.   Menampilkan Metode pembayaran dan hasil input dari data pengirim yang telah diisi oleh user

 

14.   Jenis Metode Pembayaran yaitu : Bayar ditempat dan Transfer

15.   User memilih metode pembayaran dengan bayar ditempat

 

 

16.   Menampilkan pesan yang berisi Pembayaran dilakukan ketika pesana telah ditempat

17.   User memilih metode pembayaran dengan Transfer

 

 

18.   Menampilkan pesan yang berisi pemberitahuan nomor rekening dan Bank yang akan di transfer

 

19.   Menampilkan form untuk mengunggah bukti pembayaran

20.   User mengunggah bukti pembayaran

 

 

21.   Jika unggahan berhasil, sistem akan menampilkan pesan bahwa Pengiriman sedang di proses

A Final Summary

Yang telah kami dapat dari LH 2a ini yaitu, rancangan final yang disimpulkan dari hasil survey mengenai 4 rancangan, yaitu :

User ingin rancangan yang akan dibuat itu secara detail sesuai topik dan tahapan yang nyata, dengan fitur-fitur yang telah disarankan, seperti :

- fitur searching agar memudahkan user mencari makanan ataupun minuman dengan cepat tanpa harus melihat-lihat

- fitur paket makan agar user bisa mengeluarkan uang dengan harga yang lebih murah, jika dibandingkan dengan dengan memesan yang tidak menggunakan paket.

- user menyarankan agar contact person pengantar (kurir) dimasukkan kedalam rancangan, agar memudahkan user ketika pemesanan terjadi keterlamabatan, jadi bisa mengkonfirmasi.
