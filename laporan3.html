Laporan Hasil 3

Area Permasalahan dan Rancangan

Area permasalahan dari projek ini adalah bagaimana cara memberikan interface yang mudah bagi user (pembeli di lab bisnis AN) untuk memesan makanan/minuman sesuai dengan keingingannya. Sesuai dengan permasalahan tersebut user menginginkan adanya sistem pemesanan yang mempermudah user dalam memilih makanan/minuman yang diinginkan.

Melihat area permasalahan tersebut, maka kami membuat sebuah rancangan aplikasi berbasis website yang memfasilitasi user dalam memilih makanan/minuman yang diinginkan. Untuk memudahkan user dalam mencari produk favorit, kami merancang tampilan rating dalam setiap menu makanan/minuman yang disediakan.

Usability Goals

  • Effectiveness (how good a product is at doing what it is supposed to do)
    • Apakah user dapat menemukan makanan/minuman yang diinginkan?
    • Apakah user dapat melakukan pemesanan produk sampai akhir task?
  • Efficiency (the way a system supports users in carrying out their tasks)
    • Apakah seluruh task dapat dilakukan dengan cepat?
    • Apakah ada eksekusi task yang justru akan lebih cepat dieksekusi secara manual (tanpa bantuan aplikasi)?
  • Safety (protecting the users from dangerous conditions and undesirable situations)
    • Apakah user melakukan banyak kesalahan ketika berinteraksi dengan sistem?
  • Utility (extent to which the system provides the right kind of functionality so that users can do what they need or want to do)
    • Apakah aplikasi sudah memberikan fitur yang dibutuhkan?
  • Learnability (how easy a system is to learn to use)
    • Bagaimana performa user ketika pertama kali mengerjakan task pada aplikasi?

Teknik Evaluasi

Pada evaluasi kali ini kami menggunakan tiga teknik, yaitu:

  • Observasi dan monitoring
  • Opini user
  • Key Stroke Level Model

Keterlibatan User

Pada evaluasi kali ini, user terlibat dalam:

  • Memberikan kritik dan juga saran terhadap prototipe sistem yang dibuat
  • Mencoba prototipe sistem yang dibuat

Task yang harus dikerjakan user

Ada beberapa task yang harus dikerjakan user:

  • Memilih menu makanan/minuman
  • Melihat detail makanan/minuman
  • Memilih metode pembayaran
  • tambahkan kalau kurang

Analisis dan Penjelasan

  • Halaman Utama

    • Dari hasil analisis di halaman utama ini masih banyak yang harus diperbaiki contoh nya pada fitur kategori, user masih bingung tentang letak dan juga bentuk yang seperti itu. Kemudian di fitur keranjang user bahkan tidak mengetahui kalau ada fitur tersebut Karena peletakan dan juga tidak ada tambahan gambar atau icon yang dapat menunjang keberadaan fitur tersebut. Dari penjelasan tersebut, dapat disimpulkan pula bahwa Halaman utama tidak memenuhi dua dari lima usability kriteria, yaitu tingkat kesalahan rendah dan kepuasan user secara subjektif.

  • Halaman Detail Makanan

    • Dari hasil analisis di halaman utama ini masih banyak yang harus diperbaiki contoh nya pada fitur kategori, user masih bingung tentang letak dan juga bentuk yang seperti itu. Kemudian di fitur keranjang user bahkan tidak mengetahui kalau ada fitur tersebut Karena peletakan dan juga tidak ada tambahan gambar atau icon yang dapat menunjang keberadaan fitur tersebut. Dari penjelasan tersebut, dapat disimpulkan pula bahwa Halaman utama tidak memenuhi dua dari lima usability kriteria, yaitu tingkat kesalahan rendah dan kepuasan user secara subjektif.

  • Halaman Pemesanan

    • Dari hasil analisis untuk halaman pemesanan ini sangat detail terdapat informasi tentang pembeli dan juga status dan cara pembayaran serta pengiriman. Dalam detail pengiriman terdapat menu apa saja yang dipesan dan jumlah yang harus dibayar oleh pembeli. Di task ini user tidak mendapat kesulitan dan mudah dimengerti Hal ini disebabkan karena tampilan halaman ini cukup sederhana sehingga memudahkan user. Dapat disimpulkan di task ini memenuhi aspek Efficiency, Utility dan juga Safety.

  • Halaman Data Pengiriman

    • Dari hasil analisis untuk halaman data pengiriman ini terdapat metode pengiriman yang berisi cara pengiriman dimana ada 2 cara yaitu diantarkan atau mengambil sendiri. Selain itu, terdapat lokasi pengambilan pesanan apabila ingin mengambil langsung. User yang menggunakan ini pertama kali dapat menggunakan nya tanpa harus bertanya terlebih dahulu. Karena kemudahan yang didapatkan dan tampilan yang simple. Dapat disimpulkan bahwa task ini memenuhi aspek Efficiency,Utility, Safety dan juga Learnability.

  • Umum

    • Dari 2 user yang kami minta untuk mencoba protoype kami, kami mendapatkan hasil sebagai berikut:

    • Wili Nuryusron: 01 menit 47 detik (107 detik)
    • Meidiana Rizki: 02 menit 5 detik (125 detik)
    • Riani Afsari: 02 menit 3 detik (123 detik)

    Dengan menerapkan model KLSM kami memprediksi waktu yang perlu digunakan untuk menyelesaikan task, dengan rincian sebagai berikut:

    • tH: 0.4 sec
    • tP: 1.2 sec
    • tB: 0.1 sec
    • tK: 0.2 sec
    • tR: 1.24
    • tM: 1.2

    Berdasarkan metode tersebut kami menghitung waktu yang perlu digunakan user untuk menyelesaikan task yaitu 38,24 detik. Berdasarkan perbandingan waktu perkiraan dan waktu yang digunakan user untuk menyelesaikan task, maka user tidak mampu menyelesaikan task sesuai waktu yang kami prediksi. User terkendala memahami rancangan yang dibuat. Sehingga waktu untuk menyelesaikan task lebih banyak digunakan untuk memahami rancangan.

Implikasi Hasil Terhadap Rancangan

Dari hasil analisis dan juga evaluasi terhadap rancangan yang dibuat, sudah bisa membantu dan menyelesaikan masalah yang dihadapi user. Meskipun, ada beberapa yang tidak sesuai keinginan namun hal itu tidak terlalu berpengaruh dalam rencana rancangan awal yang dibuat. Sehingga bisa disimpulkan rancangan yang dibuat sudah memenuhi apa keinginan user dan dapat menyelesaikan permasalahan yang ada.

Kritik terhadap Rencana Evaluasi

  • Halaman Utama

    • Memperbaiki fitur kategori, user masih bingung tentang letak dan juga bentuk rancangan. User bahkan tidak mengetahui jika ada fitur keranjang belanja. Hal tersebut dikarenakan peletakan dan tidak ada tambahan gambar atau icon yang dapat menunjang keberadaan fitur tersebut. Dengan adanya gambar atau icon diharapkan user lebih menyadari akan adanya fitur tersebut.

  • Halaman Detail Makanan

    • Menambahkan beberapa tampilan pelengkap yang bisa membuat tampilan lebih menarik.

  • Halaman Pemesanan

    • Dihalaman ini harus ada perbaikan jenis font dan juga ukuran nya. Karena ada user yang menyukai dari ukuran dan jenis font nya.

  • Halaman Data Pengiriman

    • Lokasi pengambilan pesanan harus lebih lengkap. Kemudian ada tulisan atau petunjuk yang menyebutkan bahwa bagian tersebut merupakan informasi lokasi pengambilan pemesanan. User lebih menekankan perbaikan tampilan dan tata letak.
